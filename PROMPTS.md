# PROMPTS — 실제로 사용한 프롬프트 예시

> 이 문서는 책 『나의 25년을 AI에게 가르치다』의 참고 자료입니다.
> 본문 6장 "프롬프트가 곧 스킬의 역사였다"와 부록 E에서 다룬, 스킬에게 실제로 건넨 프롬프트의 예시 모음입니다.

---

## 이 문서는 무엇인가

이 스킬은 복잡한 명령어 대신 자연어로 작동합니다. `--type`, `--mode` 같은 옵션을 몰라도, "지금 내가 하는 업무"를 평소 말하듯 요청하면 됩니다. 같은 파일을 첨부해도, 프롬프트에 따라 다른 산출물이 나옵니다.

아래는 갈래별로 정리한 **실제 프롬프트 예시**입니다. 스킬이 내부에서 어떻게 처리하는지(동작 순서)는 생략하고, **"이럴 때 이렇게 말하면 된다"**는 예시만 담았습니다.

처음에는 열아홉 가지 업무로 시작해, 새 필요가 생길 때마다 더해져 일흔 가지가 넘는 업무로 자랐습니다. 이 목록이 늘어 온 과정이 곧 스킬이 자라 온 역사입니다. (책 6장·부록 E 참조)

각 예시는 ITU-T 표준화 실무에 특화되어 있어 낯설 수 있습니다. 중요한 것은 개별 내용이 아니라, **반복되는 업무를 자연어로 어떻게 표현했는가**라는 방식입니다.

> 참고: 스킬 본체(SKILL.md, 템플릿, 코드)는 ITU 공식 문서의 내용을 포함하므로 공개하지 않습니다. 이 프롬프트 예시는 업무를 가리키는 표현일 뿐 ITU 문서 내용 자체가 아니기에 공개합니다.

## A. 기고서 개발

### A-1. 아이디어 → 기고서 초안

```
"AI 에이전트 상호운용성 표준화 기고서를 쓰고 싶어.
 problem statement, gap analysis, typical use case를
 먼저 정리해주고, SG13 기고서 초안을 만들어줘."
```

### A-2. 기존 draft Recommendation/Supplement 수정 제안 기고서

```
"첨부한 draft Y.3541의 clause 6.2가 AI 에이전트 기능 요구사항을
 충분히 다루지 못하고 있어. 이 내용으로 수정 제안 기고서 만들어줘.
 수정되는 부분은 revision mark로 표시해줘."
```

### A-3. 신규 NP 제안 패키지

```
"AI 에이전트 간 상호운용성 프레임워크를 신규 NP로 제안하려고 해.
 problem statement, gap analysis, use case, expected deliverable을
 먼저 정리해주고, 기고서 + A.1 justification + initial draft 구성안을 만들어줘."
```

### A-4. 기고서 동료 검토

```
"첨부한 기고서 초안을 동료 검토 관점에서 봐줘.
```
